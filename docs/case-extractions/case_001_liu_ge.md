# 案件1：刘格被杀案

> 章节范围：第1-19章（约1870行）
> 难度：★（入门级，教学案件）
> 核心知识域：足迹分析、指纹痕检、反侦察识别、逻辑推理

## 一、完整因果链

### 1. 背景关系网

```
关系类型：经济利益 + 同学关系

陈益 ←(大学同学/好友/借过30万)→ 刘格(死者)
刘格 ←(大学同学/公司有业务往来)→ 范中林(凶手)
陈益 ←(间接认识)→ 范中林

关键背景：
- 刘格开软件公司，一度经营困难（入不敷出，工资发不出）
- 向陈益借了30万，但真实用途是购买"核心代码/技术"
- 卖技术的人 = 范中林，计算机科学天赋型选手，但社会发展不顺
- 30万只是定金/预付款，刘格承诺了更多利益分成
- 公司因此起死回生，接了好几个大单
- 但刘格赚了钱后反悔，不愿兑现承诺
```

### 2. 矛盾积累

```
范中林的处境：
- 大学时技术天才，但性格内向、记仇
- 毕业后怀才不遇，社会发展不如意
- 将核心技术卖给刘格，换取了30万定金+未来利益承诺
- 双方基于信任，没签合同（熟人之间的口头承诺）
- 刘格用技术让公司翻身后，食言不给后续利益

核心矛盾：
- 没有合同 → 法律途径无效
- 自己的核心劳动成果 → 被前朋友白嫖
- 性格记仇 → 积怨已久
- "从来不报仇，但被逼急了什么都干得出来"
```

### 3. 触发事件

```
案发前几天：
- 通话记录显示范中林和刘格频繁长时间通话（最长30分钟）
- 推测：最后的利益谈判破裂
- 范中林决定杀人
```

### 4. 犯罪决策与手段选择

```
凶手能力分析：
- 计算机科学专业 → 精通网络技术
- 可以通过技术手段入侵/破坏监控系统

手段选择推导：
- 需要进入刘格家 → 作为熟人可以正常上门
- 需要处理监控 → 通过网络手段破坏电梯口外走廊监控（内部攻击）
- 凶器选择 → 这里出现了关键诡计：

关键诡计（栽赃设计）：
- 范中林不是随便杀人 → 他发现（可能通过非法安装在刘格家的监控）
  陈益之前来过，用水果刀切过橙子
- 范中林决定：用同一把水果刀杀刘格
  → 凶器上留有陈益的指纹
  → 警方会第一时间怀疑陈益
  → 为自己争取时间

执行时选择：
- 戴手套或手指涂胶水（凶器上无第三方指纹）
- 在陈益离开后不久进入刘格家
- 用水果刀杀害刘格
- 留下了自己的脚印（这是失误/无法避免）
```

### 5. 执行过程与产生的物证

```
物证清单：
① 凶器（水果刀）→ 只有刘格+陈益的指纹（范中林戴手套）
② 第三人脚印 → 范中林的（被足迹分析证实在陈益之后进入）
③ 刘格家监控被破坏 → 通过网络技术手段（内部攻击）
④ 橙子果皮 → 垃圾桶内（证实陈益确实只是切橙子）
⑤ 通话记录 → 案发前范中林与刘格频繁长通话
⑥ 财务关系 → 30万的用途（购买技术/代码）
⑦ 非法监控设备（可能存在于刘格家中）→ 范中林用来监视陈益的行为
```

### 6. 反侦察行为

```
范中林做了：
✓ 戴手套 → 凶器无己方指纹
✓ 网络攻击破坏走廊监控 → 出入不被拍到
✓ 利用陈益的指纹 → 栽赃
✓ 可能在刘格家安装非法监控 → 掌握陈益的行为细节

范中林没做到/遗漏：
✗ 无法消除自己的脚印 → 现场留有第三人足迹
✗ 脚印重叠分析 → 证明第三人在陈益之后进入
✗ 通话记录暴露 → 案发前频繁联系
✗ 性格缺陷 → 审讯时心理素质一般，较快认罪
```

### 7. 暴露点（破案关键）

```
暴露点A（物理痕迹）：
  第三人脚印 + 足迹重叠分析
  → 发现条件：痕检技术科做重叠脚印分析
  → 知识域：足迹分析/痕检
  → 作用：排除陈益嫌疑，确认有第三人

暴露点B（逻辑矛盾）：
  动机方向反转 — 陈益是债权人（刘格欠他钱），杀债务人对他没好处
  → 发现条件：基本逻辑推理
  → 知识域：犯罪动机分析
  → 作用：初步排除陈益

暴露点C（技术线索）：
  监控被内部网络手段破坏 → 凶手懂技术 → 刘格开软件公司
  → 发现条件：技术鉴定 + 关联推理
  → 知识域：技侦/IT
  → 作用：缩小嫌疑人范围到"懂计算机技术的熟人"

暴露点D（人际+财务）：
  30万的真实用途 → 购买技术 → 卖方就是范中林 → 利益纠纷
  → 发现条件：了解刘格的经营状况 + 通话记录锁定频繁联系人
  → 知识域：经侦/人际调查
  → 作用：直接指向嫌疑人

暴露点E（心理）：
  范中林"内向记仇，被逼急什么都干得出来"的性格
  → 发现条件：心理分析 + 人际关系调查
  → 知识域：犯罪心理学
  → 作用：佐证嫌疑人画像
```

## 二、游戏化设计要点

### 属性检定映射

| 调查行动 | 所需属性 | 阈值 | 获得信息 |
|---------|---------|------|---------|
| 现场足迹重叠分析 | 鉴定学 3+ | 低 | 第三人脚印在陈益之后 |
| 发现监控破坏方式 | 技术力 3+ | 低 | 内部网络攻击 |
| 查通话记录 | 调查(基础行动) | — | 范中林频繁长通话 |
| 理解30万用途 | 知识(IT行业) 或 魅力(问熟人) | 中 | 购买核心技术 |
| 推断"未签合同"意味着什么 | 心理学 3+ | 中 | 熟人关系导致无书面约定 |
| 发现非法监控设备 | 观察力 5+ / 技术力 5+ | 高(加分项) | 凶手监视了陈益行为 |

### 时间消耗设计

| 行动 | 耗时 | 重要性 |
|------|------|--------|
| 去现场勘查 | 2h | 必要（获取脚印等物证） |
| 查通话记录 | 1h | 必要（锁定嫌疑人） |
| 了解刘格公司情况 | 2h | 重要（理解动机） |
| 调查监控破坏方式 | 1h | 有帮助（缩小范围） |
| 走访刘格同事/朋友 | 2h | 有帮助（获取背景信息） |
| 搜查非法监控 | 1h | 加分项（证实栽赃） |

### 红鲱鱼

```
红鲱鱼1：陈益自己
  - 凶器有他指纹
  - 有"经济纠纷"
  - 案发前到过现场
  → 排除方式：动机方向反转(他是债权人) + 脚印时序

红鲱鱼2：刘格的其他商业对手
  - 软件公司有竞争对手
  → 排除方式：监控破坏方式指向"内部人员/技术人员"

红鲱鱼3：刘格的感情关系
  → 排除方式：现场无财物丢失、无翻动痕迹 → 熟人仇杀
```

### 推理板设计

```
核心推理链（玩家需要连接）：

[凶器只有两人指纹] + [第三人脚印存在且在后]
    → "凶手戴了手套，在陈益之后进入"

[监控被网络技术手段破坏] + [刘格开软件公司]
    → "凶手是懂计算机的人，可能与公司有关"

[30万用途是买技术] + [公司起死回生] + [没签合同]
    → "技术卖方是熟人，且利益分配有问题"

[通话记录中范中林频繁长通话] + [范中林是计算机高手]
    → "范中林最有嫌疑"
```

## 四、因果链完整性校验

| 检查项 | 结果 |
|-------|------|
| 动机是否合理？ | ✅ 核心技术被白嫖，口头承诺被违背，无法律途径 |
| 手法是否匹配凶手能力？ | ✅ IT技术人员 → 能网络攻击监控；内向记仇 → 蓄谋型 |
| 物证是否匹配手法？ | ✅ 刀伤+无第三方指纹+第三人脚印 |
| 反侦察是否匹配凶手智商？ | ✅ 高智商但非专业 → 能想到手套/破坏监控，但忽略了脚印和通话记录 |
| 暴露点是否≥2个独立的？ | ✅ 脚印(物理) + 通话记录(信息) + 动机反转(逻辑) |
| 时间线是否自洽？ | ✅ 陈益先到→离开→范中林后到→杀人→离开 |
| 是否存在"为什么不用更简单方法"的漏洞？ | ✅ 选择水果刀(栽赃)而非自带凶器，有合理解释 |
