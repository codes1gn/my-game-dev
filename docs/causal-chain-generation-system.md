# 因果链驱动的案件生成系统设计

> 状态：设计中（2026-06-02）
> 依赖：`docs/case-extractions/` 23个案件因果链文档
> 前置：`docs/roguelike-investigation-design.md`, `docs/material-library-design.md`

---

## 一、核心理念

**不是随机拼凑犯罪元素，而是生成一条完整的因果链。**

传统案件生成 = 随机选凶器 + 随机选动机 + 随机选场景 → 拼不出逻辑自洽的故事
因果链生成 = 从"人物背景+矛盾积累"出发 → 自然推导出犯罪行为 → 反向生成暴露点

```
[背景关系网] → [矛盾积累] → [触发事件] → [犯罪决策] → [手段选择] → [执行过程] → [善后行为] → [暴露点]
     ↑                                                                                    ↑
   输入层：NPC模板 + 关系网                                              输出层：玩家可调查的线索
```

## 三、因果链生成算法

### 3.1 生成流程

```
Step 1: 选择背景关系网模板 (R01-R14)
        → 生成 2-6 个NPC + 关系
        
Step 2: 选择矛盾积累模式 (M01-M10, 可叠加2个)
        → 注入NPC之间的张力
        → 校验矛盾与关系网的逻辑兼容性
        
Step 3: 生成触发事件
        → 从矛盾模式推导出"最后一根稻草"
        → 触发事件必须与矛盾直接相关
        
Step 4: 犯罪决策（凶手选择）
        → 根据凶手性格+智商+职业选择手法
        → 从手法库(W01-W15)选择兼容手法
        → 手法难度 ≤ 凶手能力值
        
Step 5: 执行过程 + 善后行为
        → 生成完整时间线
        → 每个NPC在每个时间点有明确位置和行为
        
Step 6: 反向生成暴露点
        → 根据选中的手法 → 必然产生特定物证
        → 根据善后行为 → 计算"完美程度"
        → 暴露点数量 = 基础3 + (10 - 凶手智商) + 难度修正
        → 每个暴露点关联到一种调查方法(E01-E12)
        
Step 7: 兼容性校验
        → 因果链必须完整（每一步都有前因后果）
        → 暴露点必须可被玩家发现（至少3条独立路径到达真相）
        → 红鲱鱼(dry herring)不能完全堵死正确推理路径
```

## 四、Roguelike集成设计

### 4.1 案件难度与玩家等级

| 玩家等级 | 案件复杂度 | 可用因果链层数 | NPC数 | 暴露点数 |
|---------|----------|-------------|-------|---------|
| 见习（区级） | ★-★★ | 6步基础链 | 3-5 | 5-8 |
| 正式（市级） | ★★-★★★ | 7步+红鲱鱼 | 5-8 | 4-6 |
| 精英（省级） | ★★★-★★★★ | 8步+双案联动 | 8-12 | 3-5 |
| 传奇（部级） | ★★★★★ | 9步+跨城+教唆链 | 12+ | 2-4 |

### 4.2 天赋/道具对破案的影响

```
暴露点[E07_专业知识] 被发现的概率:
  base = 30%
  + 玩家鉴定学属性 * 5%
  + 如果拥有"百科全书"天赋: +20%
  + 如果使用"专家咨询"道具: +30%
  + 如果之前破过同类型案件: +10%
  
  final = min(95%, base + bonuses)
```

### 4.3 构筑爽点反馈

当天赋/道具触发成功时的显式反馈：

```
[系统提示] 🔍 天赋【微表情识别】触发！
  → 你注意到嫌疑人提到"平安符"时瞳孔骤缩，右手不自觉握拳
  → 判断：此人对平安符有强烈情感反应
  → 新线索已解锁：[嫌疑人与平安符的关联]
```

```

  → NPC: 村姑A(被害), 村民B(老实), 村民C+D(混混), 外来女E(幕后)

Step 7: 校验 ✓ 因果完整, ✓ 3条独立路径, ✓ 玩家可解
```

## 七、下一步实现路线

1. **数据层**：将23个案件结构化为JSON模板库
2. **生成器**：GDScript实现因果链随机生成算法
3. **校验器**：确保生成的案件逻辑自洽
4. **难度调节**：根据玩家等级动态调整复杂度
5. **叙事包装**：将数据模型转化为可阅读的案件描述文本
6. **测试**：生成100个案件 → 人工审核逻辑自洽性
